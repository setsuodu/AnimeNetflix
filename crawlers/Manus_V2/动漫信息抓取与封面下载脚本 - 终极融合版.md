# 动漫信息抓取与封面下载脚本 - 终极融合版

## 概述

本脚本旨在高效、准确地从多个在线源（Wikipedia、Bangumi、AniList）抓取动漫相关信息，并下载封面图片。它融合了高并发处理、实时保存、精细化日志记录、多级标题清洗与模糊匹配，以及最权威的 Wikipedia 跨语言标题对齐机制，旨在提供“准 Gemini 级”的准确率和高效率。

## 主要特性

*   **三级火箭搜索策略**：
    1.  **Wikipedia (定名核心)**：优先通过维基百科的 `langlinks` 获取最权威的日文原名和英文译名。同时，会尝试从页面内容中解析出罗马字或官方英文译名，作为 `title_EN` 的最高优先级。
    2.  **Bangumi (补图与社区化命名)**：在 Wikipedia 未能提供完整信息（特别是高质量封面）时，作为补充来源。利用其丰富的社区数据和别名库进行模糊匹配。
    3.  **AniList (全球化兜底)**：在前两个来源都无法提供完整信息时，作为最终的补充来源，利用其高质量的英文标题和封面。
*   **信息融合逻辑**：不同来源获取到的信息会智能合并，取长补短，确保最终数据的完整性。
*   **高效率并发处理**：采用 3 线程并发，显著提升数据抓取速度，避免长时间等待 API 响应。
*   **实时保存与断点续传**：每处理完一条数据即刻保存，并支持从上次中断处继续运行，确保数据安全不丢失。
*   **精细化日志系统**：详细记录每条数据的处理状态，包括成功、未找到、超时、频率限制、API 错误等，方便用户追踪和排查问题。
*   **多级标题清洗与模糊匹配**：包括“去季”、“短标题”和“更激进的短标题”重试策略，并引入 `difflib` 进行字符串相似度计算，提高匹配成功率。
*   **纯 ASCII 文件名策略**：确保下载的封面图片文件名是纯英文或拼音，以兼容 NAS、Nginx 等文件系统和 Web 服务。

## 文件名生成策略 (纯 ASCII)

为了确保文件名对 NAS 和 Nginx 友好，本脚本采用了严格的纯 ASCII 文件名生成策略，优先级如下：

1.  **Wikipedia 解析出的罗马字/官方英文译名**：这是最高优先级，脚本会尝试从维基百科页面内容中提取最官方的英文或罗马字名称。
2.  **Wikipedia `langlinks` 提供的英文标题**：如果未能从页面内容中解析出，则使用 `langlinks` 提供的英文标题。
3.  **AniList 提供的英文标题**：如果 Wikipedia 未能提供合适的英文标题，则尝试使用 AniList 的英文标题。
4.  **Bangumi 提供的英文标题**：如果 Bangumi 有提供英文标题，则使用。
5.  **原始中文标题的拼音**：作为最终的兜底方案，如果以上所有英文标题都无法获取，脚本会将原始中文标题转换为拼音作为文件名。这通过 `pypinyin` 库实现。

**日文标题将不再直接作为文件名选项**，以避免非 ASCII 字符带来的兼容性问题。

## 使用方法

1.  **安装依赖**：
    ```bash
    pip install requests tqdm pypinyin beautifulsoup4
    ```
2.  **准备输入文件**：
    将您的动漫列表保存为 `anime_list.json`，格式应为 JSON 数组，每个元素包含一个 `"title"` 字段，例如：
    ```json
    [
        {"title": "18if 梦境异闻录"},
        {"title": "宝可梦"},
        {"title": "某科学的超电磁炮S"}
    ]
    ```
3.  **运行脚本**：
    ```bash
    python super_anime_optimized.py
    ```

## 输出文件

*   `anime_list_final_fixed.json`：处理后的动漫列表，包含 `title_JP`, `title_EN`, `coverFile`, `coverURL` 等信息。
*   `covers/`：下载的封面图片将保存在此文件夹中，文件名将遵循纯 ASCII 策略。
*   `crawl_log.txt`：详细的运行日志，记录每条数据的处理状态和错误信息。
*   `failed_to_manual_search.json`：未能成功处理的条目列表，方便手动排查。

## 日志级别说明

*   `[START]`：脚本启动信息。
*   `[INFO]`：一般信息，如加载数据、进度更新。
*   `[PROCESS]`：开始处理单个条目。
*   `[CLEAN]`：标题清洗后的结果。
*   `[TRY]`：尝试不同搜索策略（去季、短标题）。
*   `[DEBUG]`：详细的调试信息，如 API 命中。
*   `[SUCCESS]`：成功处理一个条目。
*   `[NOT_FOUND]`：API 响应正常，但确实没有匹配结果。
*   `[TIMEOUT]`：请求超时。
*   `[RATE_LIMITED]`：触发 API 频率限制。
*   `[API_ERROR]`：API 返回非 200 错误码。
*   `[JSON_ERROR]`：API 返回数据解析失败。
*   `[ERROR]`：其他未知错误。
